Detecting anomalous order flow with a deep autoencoder

Trains a bottleneck neural network on limit order book features — bid/ask returns, queue sizes, trade and cancellation indicators — then uses reconstruction error as an unsupervised anomaly score. No labels required: the model learns what "normal" order flow looks like and flags everything that doesn't compress cleanly.

The bottleneck forces the model to learn a compressed representation of normal order flow. Events that cannot be compressed accurately — unusual order sizes, atypical bid/ask dynamics, abnormal cancellation bursts — produce high reconstruction error and are flagged as anomalies. Dropout (0.2) at each layer prevents overfitting to noise; BatchNorm stabilises training on the heterogeneous LOB feature scales.

Scores are normalised to [0, 1] per the submission format. Submitted as raw continuous scores (not binary threshold) — better for AUC-ROC evaluation since it preserves the full ranking information rather than collapsing it to a single cut-off.
Pipeline
End-to-end methodology
1
Feature extraction & alignment
Parse raw LOB messages into per-symbol feature rows. Construct SeqRepeatCount by groupby on symbol × sequence number — detects duplicate/replayed messages which are themselves anomaly-adjacent.
2
Preprocessing
StandardScaler fit on train only, applied to val and test. Missing values in test imputed with train means — no data leakage. All 15 features standardised to zero-mean unit-variance before the network sees them.
3
Autoencoder training
15 → 64 → 32 → 4 → 32 → 64 → 15 with BatchNorm + ELU + Dropout(0.2). MSE reconstruction loss. Best checkpoint saved by validation loss; cosine LR annealing prevents oscillation late in training.
4
Anomaly scoring
Per-sample MSE between original and reconstructed features. Events that deviate from the distribution the autoencoder learned — unusual order sizes, atypical cancellations — receive high error scores.
5
Normalisation & submission
Min-max normalise errors to [0,1]. Submit as continuous scores for AUC-ROC evaluation — preserves ranking information that binary thresholding would discard.
Core model
Autoencoder architecture
class Autoencoder(nn.Module): def __init__(self, input_dim, bottleneck=4): super().__init__() self.encoder = nn.Sequential( nn.Linear(input_dim, 64), nn.BatchNorm1d(64), nn.ELU(), nn.Dropout(0.2), nn.Linear(64, 32), nn.BatchNorm1d(32), nn.ELU(), nn.Dropout(0.2), nn.Linear(32, bottleneck) # ← compress to 4-dim ) self.decoder = nn.Sequential( nn.Linear(bottleneck, 32), nn.BatchNorm1d(32), nn.ELU(), nn.Dropout(0.2), nn.Linear(32, 64), nn.BatchNorm1d(64), nn.ELU(), nn.Dropout(0.2), nn.Linear(64, input_dim) # ← reconstruct ) def forward(self, x): return self.decoder(self.encoder(x)) # Anomaly score = per-sample reconstruction MSE errors = torch.mean((recon - x) ** 2, dim=1) scores = (errors - errors.min()) / (errors.max() - errors.min())
Why ELU over ReLU? ELU has non-zero gradient for negative inputs, which helps with the normalised LOB features that are frequently negative (e.g. negative bid returns during sell pressure). BatchNorm before activation stabilises the dynamic range across heterogeneous feature types.
