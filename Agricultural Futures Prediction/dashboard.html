Explainability

The three sources have mismatched frequencies: USDA is weekly, NOAA is monthly, futures are daily. All data is resampled to the last trading day of each month. USDA reports are shifted 1 period forward (released Monday, used the following month) and PDSI is lagged 1 month — both to ensure zero look-ahead bias.

Feature engineering
13 engineered features across 3 categories
☀
pdsi_3m — 3-month rolling PDSI avgnovel
Cumulative soil moisture deficit affects yield more than a single month spike. Smooths noise while preserving drought signal.
☀
pdsi_velocity — month-on-month changenovel
Markets react to the direction of drought, not just its level. A drought intensifying from −2 to −4 is a different signal than a stable −3.
☀
extreme_drought — binary flag (PDSI < −3)
Historical analysis shows prices spike non-linearly past this threshold. A linear PDSI value understates the effect at extremes.
🌽
condition_idx — weighted condition scorenovel
Formula: 2×Excellent + 1×Good − 1×Poor − 2×Very Poor. Weights extremes more than middle — mirrors how traders read the Monday USDA report.
🌽
condition_z — seasonal z-score deviation
Z-score of condition_idx vs. same-calendar-month 5-year expanding average. Markets price surprise, not absolute level — removes seasonal baseline entirely.
Key differentiator: These cross-domain interaction features are what separate this project from a pure quant exercise. Neither PDSI nor crop condition alone captures the compound effect — the interaction term is the #1 SHAP predictor in the model.
⚡
pdsi_x_condition — PDSI × condition index★★ key
Drought reported by NOAA and poor crop condition reported by USDA compound non-linearly. A linear model treats them as additive; the product term captures the true multiplicative stress on supply.
⚡
drought_critical — drought onset during pollination★★ key
PDSI velocity < −1 during July–August (corn pollination window). The same drought level has ~3× the yield impact during pollination vs. September — timing is everything.
📈
mom_1m, mom_3m, mom_12m — price momentum
Log return over 1, 3, and 12 months. Baseline price-based signals for comparison against the alternative data features.
📈
price_to_5yr — mean-reversion signal
Current price divided by 60-month rolling average. Captures whether corn is historically cheap or expensive — a fundamental anchor.
📈
vol_3m — realised volatility
3-month rolling standard deviation of log returns. Regime indicator — high vol often coincides with supply shocks.

Methodology
Critical design decisions
1
No look-ahead bias: USDA weekly condition reports shifted 1 period — released Monday, not available until the following month's feature set. PDSI lagged 1 month to simulate real-time NOAA availability.
2
Expanding-window CV: Train on all history up to month t, predict month t+1. Expanding (not rolling) retains the 2012 drought event in training history once it passes.
3
Seasonal z-scores: All rolling statistics computed within same-calendar-month cohorts — avoids cross-season contamination that would inflate apparent signal.
4
Regime robustness: Model trained pre-2012 correctly signals the historic drought-driven price spike — a clean out-of-distribution validation that the features generalise.

Quickstart
Run the pipeline

No API keys required to run on synthetic demo data. Set use_synthetic=False and add keys to use real USDA / NOAA / Nasdaq Data Link data.

# Install dependencies pip install pandas numpy requests \ scikit-learn xgboost shap \ matplotlib nasdaqdatalink # Run (synthetic demo data) python agri_futures_pipeline.py # Run with real data export USDA_KEY="your_usda_key" export NOAA_TOKEN="your_noaa_token" export NASDAQ_DL_KEY="your_ndl_key" python agri_futures_pipeline.py
